# 第二周：面向对象编程

## 学习目标
系统学习C++面向对象编程的核心概念，包括类与对象、继承、多态、运算符重载、模板和异常处理，为后续的STL和高级特性学习打下坚实基础。

## 学习计划

### 第8天：类与对象基础
- **学习内容**：类的定义与声明、对象的创建与销毁、成员变量与成员函数、访问控制、this指针
- **实践重点**：银行账户类实现、成员函数设计、访问控制应用
- **算法题**：LRU缓存、设计循环队列、设计哈希集合、设计哈希映射

### 第9天：构造函数与析构函数
- **学习内容**：构造函数定义与调用、析构函数作用、拷贝构造函数、移动构造函数、初始化列表
- **实践重点**：资源管理类实现、RAII原则、异常安全设计
- **算法题**：设计推特、设计前K个高频元素、设计循环双端队列、设计一个支持增量操作的栈

### 第10天：继承
- **学习内容**：基类与派生类、继承方式、派生类构造与析构、函数重写、多重继承
- **实践重点**：图形类继承体系、继承层次设计、虚析构函数
- **算法题**：二叉搜索树的最近公共祖先、删除二叉搜索树中的节点、验证二叉搜索树、二叉搜索树中的插入操作

### 第11天：虚函数与多态
- **学习内容**：虚函数概念、虚函数表、纯虚函数与抽象类、虚析构函数、动态绑定
- **实践重点**：多态动物类体系、虚函数表理解、动态绑定机制
- **算法题**：二叉树的最近公共祖先、路径总和III、验证完全二叉树、二叉树中的最大路径和

### 第12天：运算符重载
- **学习内容**：可重载运算符、运算符重载规则、友元函数与成员函数、特殊运算符重载、类型转换运算符
- **实践重点**：复数类实现、运算符重载设计、友元函数应用
- **算法题**：基本计算器II、逆波兰表达式求值、设计计算器、字符串解码

### 第13天：模板基础
- **学习内容**：函数模板定义与使用、类模板定义与使用、模板参数、模板特化、模板实例化
- **实践重点**：通用栈类模板、模板元编程基础、模板特化应用
- **算法题**：合并K个升序链表、数组中的第K个最大元素、前K个高频元素、最接近原点的K个点

### 第14天：异常处理
- **学习内容**：try-catch-throw机制、异常类型、异常规范、RAII与异常安全、标准异常类
- **实践重点**：异常安全资源管理、异常处理最佳实践、RAII模式应用
- **算法题**：设计一个支持增量操作的栈、设计循环双端队列、设计推特、设计前K个高频元素

## 学习重点

### 1. 面向对象设计原则
- 理解封装、继承、多态三大特性
- 掌握良好的类设计原则
- 学会使用访问控制保护数据

### 2. 内存管理与RAII
- 理解构造和析构的生命周期
- 掌握RAII（资源获取即初始化）原则
- 学会设计异常安全的类

### 3. 继承与多态
- 深入理解继承机制和虚函数表
- 掌握多态的实现原理
- 学会设计合理的继承层次

### 4. 运算符重载
- 掌握运算符重载的规则和最佳实践
- 理解友元函数的使用场景
- 学会设计直观的类接口

### 5. 模板编程
- 理解模板的实例化过程
- 掌握模板特化的使用
- 学会设计通用的模板类

### 6. 异常处理
- 掌握异常处理的机制
- 理解异常安全的重要性
- 学会设计异常安全的代码

## 算法题重点

### 数据结构设计
- **LRU缓存**：双向链表 + 哈希表
- **循环队列**：数组实现循环
- **哈希表**：设计哈希函数和冲突处理
- **二叉搜索树**：BST的基本操作

### 高级算法技巧
- **设计模式**：观察者模式、策略模式
- **动态规划**：路径问题、最优化问题
- **图算法**：最近公共祖先、路径问题
- **数学计算**：表达式求值、计算器设计

### 编程技巧
- **面向对象设计**：合理的类层次结构
- **异常安全**：确保异常安全的内存管理
- **模板编程**：泛型编程思想
- **性能优化**：时间复杂度优化

## 学习建议

### 1. 每日学习流程
1. **理论学习**（30-45分钟）：阅读相关概念和语法
2. **代码实践**（45-60分钟）：完成当天的实践练习
3. **算法练习**（60-90分钟）：完成4道算法题
4. **总结复习**（15分钟）：整理笔记，总结重点

### 2. 学习方法
- **面向对象思维**：用面向对象的方式思考问题
- **设计模式**：学习常见的设计模式
- **实践为主**：每学一个概念都要写代码验证
- **代码复用**：学会设计可复用的类

### 3. 注意事项
- **内存管理**：注意构造和析构的配对
- **虚函数**：理解虚函数表的工作原理
- **异常安全**：确保代码的异常安全性
- **性能考虑**：了解不同实现的性能差异
